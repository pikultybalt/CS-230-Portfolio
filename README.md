# CS 230 Portfolio Reflection – The Gaming Room Software Design

# Project Summary

The client for this project was The Gaming Room, a company that wanted to expand its game, 
Draw It or Lose It, from a single Android application into a web-based, cross-platform application. 
The software needed to support multiple teams and players while ensuring that each game, team, and 
player had a unique name. The system also needed to support multiple operating systems and provide 
a reliable, secure, and scalable architecture that could accommodate future growth.

# What I Did Well

One of the strongest parts of my work was evaluating multiple operating platforms and recommending 
the best solution based on the client's requirements. I also documented the software architecture 
clearly and explained why specific technologies and design patterns would best support a scalable, 
secure, and maintainable application. The completed design document effectively communicated both 
the technical requirements and the reasoning behind the design decisions.

# What I Found Helpful

Working through the software design document before writing code helped organize the project's 
requirements and identify potential issues early in the development process. Creating the design 
first provided a clear roadmap for implementation and reduced the likelihood of making major changes 
later. It also made it easier to understand how the different components of the system would interact.

# What I Would Revise

If I could revise one part of my work, I would expand the discussion of security and cloud deployment. 
I would include more detailed recommendations for authentication, data encryption, load balancing, and 
disaster recovery. Adding architecture diagrams would also improve the clarity of the design and make it 
easier for developers to understand the system.

# Understanding User Needs

I interpreted the client's needs by carefully reviewing the project requirements and translating them 
into software features and technical recommendations. The design ensures that every game, team, and player 
has a unique identifier while supporting multiple users across different platforms. Considering user needs 
is essential because software should solve the client's problems while remaining easy to use, reliable, secure, 
and scalable. Meeting user expectations leads to a better overall experience and increases the likelihood of 
project success.

# My Approach to Software Design

My approach to software design begins with gathering and analyzing the client's requirements before selecting 
technologies and designing the system architecture. I break the project into smaller components, evaluate 
multiple design options, and choose solutions that balance performance, security, maintainability, and scalability. 
In future projects, I will continue using object-oriented design principles, UML diagrams, modular design, iterative 
development, and thorough requirements analysis to create high-quality software applications that meet both current 
and future business needs.
